# Apache Airflow

Инструмент, который позволяет запускать скрипты, как в ручном режиме (через CLI), по расписанию или по событию.

Для работы требует: 
* БД для сохранения метаинформации
* Планировщик (scheduler)
* Веб-сервер 

Позволяет работать через GUI (http://localhost:8080) или через консоль

В данном примере, для запуска использовался docker-compose, который позволяет сразу запустить все контейнеры, которые требуются для airflow.


## Запуски DAG-ов
Чтобы запустить созданный скрипт, нужно, чтобы он располагался в папке `./dags` - монтировали в docker-compose.yml

Запускать можно, как через UI airflow (http://localhost:8080 - в браузере),
так и через командную строку, войдя в оболочку контейнера, в котором крутится airflow (scheduler).

Чтобы войти в оболочку контейнера:
```cmd
docker exec -it container_name bash
```

Зайдя в контейнер, можно взаимодействовать с airflow через `airflow`:
```bash
airflow version # покажет версию airflow
airflow cheat-sheet # выведет подсказки
```

Например:
```bash
airflow variables list # список всех установленных переменных
airflow connections list # список всех доступных подключений (будет много)
```

Далее необходимо добавить покдлючение.
Использовав команду `airflow connections add` - выйдет подсказка, со всеми необходимыми для передачи флагами

Пример создания подключения (обязательный позиционный аргумент connection_id):
```bash
airflow connections add test_conn1 \
    --conn-type 'generic' \
    --conn-host 'minio' \
    --conn-login 'adm' \
    --conn-password 'secret_pass' \
    --conn-port 9000 \
    --conn-extra '{"endpoint_url":"http://minio:9000"}'
```

Чтобы проверить, что connection был создан, можно попробовать получить его:
```bash
airflow connections get test_conn1
```

Теперь данное подключение можно парсить с помощью BaseHook в скриптах

Еще можно добавить переменные, которые также можно будет использовать в скриптах

Добавить переменную:
```bash
airflow variables set # замечательная документация, которая объясняет, что передавать 
```

```bash
airflow variables set root_username adm # пример конкретной установки
airflow variables set root_password secret_pass
airflow variables set endpoint_url http://minio:9000
```

Посмотреть список всех переменных
```bash
airflow variables list
```

Подключения `connections` лучше использовать для хранения чувствительных данных (паролей и прочего), для общих данных, можно создавать файлы конфигураций рядом с самими dag'ами и в них определять нужные значения.

Важный момент про `connections` и `variables`, если подключение или переменные парсятся в dag'ах, которые запускаются, а данных переменных не существует, весь граф не будет выполнен.


## Запуск задач
После того, как были установлены соответствующие переменные и было создано подключение (параметры для подключения)
Можно переходить к запуску задач.

Можно запускать как целый пайплайн (весь файл, со всеми задачами, по сути выполнить весь DAG)
Можно запустить отдельную задачу (по её task_id, который устанавливался во время создания задачи)

Задачи должны быть атомарны, не стоит делать задачу, которая для выполнения требует данных от другой задачи.


### Запуск задачи (по task_id)
Для DAG в целом устаналивается ID (dag_id), по данному ID и осуществляется поиск DAG, который будет запускаться. 
Внутри DAG также просматриваются все задачи, которые были созданы и находится с желаемым (task_id).

Вот пример (spark_connect_test - dag_id, test_task_params_1 - task_id)
т.е просим, запусти вот в этом DAG c dag_id, задачу с этим task_id:
```bash
airflow tasks test simple_dag simple_print 2026-04-21
```
После запуска произойдет вывод лога в терминал

### Запуск всего DAG (по dag_id)
Команда для работы с DAG:
```bash
airflow dags # выведет все подсказки
```

Список всех dag
```bash
airflow dags list
```

Сделать dag активным
```bash
airflow dags unpause spark_connect_test
```

Запустить:
```bash
airflow dags trigger spark_connect_test
```

Просмотреть статус выполнения
```bash
airflow dags list-runs -d spark_connect_test
```
