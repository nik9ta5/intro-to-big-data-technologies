## Формат Apache Iceberg

[Документация](https://iceberg.apache.org/)  
[Что такое Apache Iceberg](https://aws.amazon.com/ru/what-is/apache-iceberg/)

**Apache Iceberg** — это открытый формат таблиц для озёр данных. Он хранит подробные метаданные, обеспечивает ACID-транзакции, версионирование и упрощает обработку больших объёмов данных.

Spark поддерживает Iceberg из коробки. Для работы нужно добавить соответствующие параметры в конфигурацию **SparkSession**.  
Полная конфигурация уже подготовлена в `./connect/spark_session.py`.

## ETL-пайплайн (Extract — Transform — Load)

Пример простого ETL-пайплайна находится в `./connect/etl_simple.py`.  
Для запуска скрипта нужно передать аргументы командной строки (парсинг реализован в `tools.py`).

**На что стоит обратить внимание:**
- Путь к источнику данных (бакету в MinIO) должен быть указан корректно.
- Нужно задать название базы данных и таблицы Iceberg, в которую будут записываться данные.
- Убедиться, что целевой бакет существует (иначе Spark не сможет создать таблицу).
- Не забыть передать при запуске имя пользователя и пароль MinIO (те, что указывались при создании).
- В конце скрипта выбрать режим записи: `createOrReplace` или `overwritePartitions` (методы отличаются).

## Data Lakehouse

**Data Lakehouse** = MinIO + Apache Iceberg + PySpark

Такое хранилище мы получаем, сочетая объектное хранилище (MinIO), табличный формат Iceberg и движок обработки PySpark.  

Теперь работа с данными осуществляется по принципам **ACID**.  
Появляется полноценное **версионирование** (time travel): в любой момент можно откатиться к любой предыдущей версии данных (пока существует snapshot).

Взаимодействовать с Lakehouse можно через Spark:
- DataFrame API
- SQL API

Пример запросов к Lakehouse — в `./connect/lakehouse_query.py`.

## HW 3


Создать свой ETL/ELT пайплайн (на основе рассмотренных в семинаре скриптов). Создать свой Data Lakehouse.


1. Взять любой датасет (можно взять из предыдушего задания, одного файла достаточно).
2. Разделить выбранный датасет на 2 части (в любом соотношении, 70/30, 50/50, не важно).
Можно не создавать новые физические файлы, просто сделать фильтрацию по определенной дате (меньше или равно, больше).
3. Создать хранилище (или использовать созданное в прошлом задании). 
4. Создать новый бакет для работы. 
5. Написать скрипт для загрузки первой части данных в хранилище (без предобработки). Сразу использовать Iceberg.
Можно использовать один каталог и одну базу данных, загружать данные в разные таблицы. 
Например, в скрипте `spark_session.py` указано имя каталога `local_catalog`.
Можно создать одну бд и в ней две таблицы, одну с необработанными данными, другую с обработанными.
6. Использовать тот же скрипт для загрузки второй части данных (в ту же таблицу, также без предобработки).
7. Написать скрипт для предобработки данных (мы загрузили грязные данные в хранилище, теперь будем обрабатывать) (пример, `etl_simple.py`)
(Название каталога, название базы данных, название таблицы)
8. Выполнить предобработку, загрузить предобработанные данные в новую таблицу в хранилище.
(В том же каталоге, в той же базе данных, указать новое имя таблицы).
9. Создать другой скрипт для запросов к хранилищу, к предобработанным данным (сделать пару выборок, фильтрацию, группировку).


Что должно получиться:
* Скрипт для загрузки данных в хранилище
* Скрипт для предобработки данных
* Скрипт для запросов к хранилищу
* Добавить файлик `README.md` с описанием, что было сделано на каждом шаге
